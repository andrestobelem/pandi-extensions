Desarrollar pi-cante con Picante

Este archivo vive en `pandi-extensions`, pero esta sección corresponde al
checkout sibling `pi-cante`. `./picante` no existe dentro de
`pandi-extensions`: antes de usarlo, cambiá de directorio a `pi-cante`.

  cd /Users/andrestobelem/ws/at/pi-cante
  ./picante status
  npm run smoke:picante
  npm run smoke:picante:tui
  ./picante

El wrapper `./picante` abre Picante desde source sobre el checkout de
`pi-cante`, carga el sibling `pandi-extensions` y usa el perfil descartable
`pi-cante/.pandi-dev/agent`. No modifica ~/.pi, ~/.picante ni ~/.pandi.

Después de cambiar código de Picante (CLI, TUI, RPC o scripts), cerrá y volvé a
ejecutar `./picante`. Antes de terminar un cambio de código, corré:

  npm run check

Si pandi-extensions no está como sibling:

  PANDI_EXTENSIONS_ROOT=/ruta/a/pandi-extensions ./picante

Desarrollar pandi-extensions con Picante

Esta sección sí se ejecuta desde `pandi-extensions`. No uses `./picante` aquí:
el wrapper disponible en este checkout es `npm run dev:picante`. Tampoco abras
`pi` a secas para el loop normal de desarrollo.

  cd /Users/andrestobelem/ws/at/pandi-extensions
  npm run dev:picante -- status
  npm run smoke:picante
  npm run smoke:picante:tui
  npm run dev:picante

Este wrapper localiza el checkout sibling de `pi-cante`, abre Picante desde
ese source y usa `pandi-extensions` como workspace real. El perfil aislado se
guarda en `pi-cante/.pandi-dev/agent`; no modifica ~/.pi, ~/.picante ni
~/.pandi.

Si pi-cante no está como sibling:

  PI_CANTE_ROOT=/ruta/a/pi-cante npm run dev:picante -- status

Después de editar una extensión, usá /reload o cerrá y volvé a ejecutar:

  npm run dev:picante

Para probar compatibilidad con Pi vanilla, usá `pi` a secas solo de forma
puntual, fuera de este loop.

Workflows globales de Claude

Desde pandi-extensions podés inspeccionar, instalar o quitar el espejo global
de Claude:

  cd /Users/andrestobelem/ws/at/pandi-extensions
  npm run sync:claude:global:status
  npm run sync:claude:global:install
  npm run sync:claude:global:remove

`status` no escribe. `install` instala o restaura los workflows, scripts y
skills gestionados en ~/.claude. `remove` quita únicamente archivos que siguen
idénticos a los que instaló Pandi; conserva archivos ajenos o modificados.

Para probar otro destino sin tocar ~/.claude:

  CLAUDE_GLOBAL_DIR=/ruta/de/prueba npm run sync:claude:global:install
